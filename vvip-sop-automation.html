VVIP 学员服务体系

SOP 全自动化设计全览

将 5 阶段 VVIP 学员服务标准流程拆解为 7 个可自动运行的 Skill 模块，实现极致跟进、专业数据沉淀、有温度的家校沟通。

一、Skill 模块总览
# Skill 名称 对应 SOP 阶段 触发方式 优先级 Carol 是否介入
第一阶段：正式课建档与启动
1 student-onboarding 正式课建档与启动 手动触发（新学员签约时） P1 ✅ 首次需人工初始化
第二阶段 + 第三阶段：提醒与日常运营
2 lesson-reminder 黄金三节课 + 日常运营 Cron 定时（每天 08:00 扫描课表） P0 ❌ 全自动
3 zoom-recorder-manager 日常运营：视频备份 Zoom Webhook（课程结束事件） P0 ❌ 全自动
4 feedback-processor 日常运营：课后反馈处理 飞书表单提交（老师填写反馈后） P1 ✅ 审核后发送
5 ai-learning-advisor 黄金三节课：AI 智能补位 由 feedback-processor 自动调用 P2 ✅ 审核资料内容
第四阶段：月度结算与进度汇报
6 monthly-reporter 月度家校沟通 Cron 定时（每月第 25 天） P2 ✅ 审核后发送
7 teacher-hours-tracker 教务核算 Cron 定时（每月最后一天） P3 ✅ 确认后报送财务
二、各 Skill 详细设计
🏗️
student-onboarding
学员建档与启动
P1
字段内容
触发方式手动触发，新学员签约后执行一次
输入数据学员姓名、年级、目标、外教/中教、上课时间、微信群 ID
动作 1调用 Zoom API → 创建学员专属固定会议链接
动作 2创建飞书文件夹：01_课前资料 / 02_课程回放 / 03_老师反馈与作业
动作 3初始化飞书多维表格学员档案（基础信息写入）
动作 4将 Zoom 链接 + 上课操作指南发送至微信服务群
动作 5分别向外教 Ryan 和中教 Vincent 发送确认消息（含上课时间）
依赖接口Zoom API、飞书 Open API、企业微信群机器人
🎥
zoom-recorder-manager
录像下载与双重备份
P0
字段内容
触发方式Zoom Webhook 事件：recording.completed
动作 1自动下载课程回放视频（MP4 格式）
动作 2上传至企业共享云盘（按学员 / 日期归档）
动作 3上传至飞书 02_课程回放 文件夹（家长随时可查）
动作 4发回放链接到微信群，附注本节课主题和时长
动作 5更新飞书多维表格课节记录，标记「已备份」
错误处理下载失败时飞书消息通知 Carol 人工处理
依赖接口Zoom Webhook + API、飞书 API、企业云盘 API、企业微信
📝
feedback-processor
反馈提炼、翻译与分发
P1
字段内容
触发方式老师提交飞书反馈表单后自动触发
AI 步骤 1提炼表扬点（具体化、价值化表述）
AI 步骤 2提炼改进点 + 解决方案（必须带方案）
AI 步骤 3"needs structure" → "需要建立应对国际学校面试的逻辑框架"
AI 步骤 4检测笼统反馈 → 若是，自动调用 ai-learning-advisor
人工介入Carol 在飞书审批流中审核，确认后点击「发送」
动作（通过后）排版后发送至微信服务群 + 归档至飞书 03_老师反馈与作业
依赖接口飞书表单 API、LLM API、飞书审批 API、企业微信
🤖
ai-learning-advisor
AI 智能补位学习建议
P2
字段内容
触发方式由 feedback-processor 检测到笼统反馈后调用
输入学员档案（年级/目标/教材进度）+ 老师笼统反馈文本
生成内容 1BBC 6分钟英语推荐（匹配本节课主题，附推荐理由）
生成内容 2Quizlet 词卡推荐 或 自动生成专属词汇表
生成内容 3阅读/听力扩展材料（匹配 G7 水平 + UWC 备考方向）
生成内容 4给学生的个性化一句话推荐语（亦师亦友的语气）
人工介入Carol 审核资料内容，确保专业准确
依赖接口LLM API、BBC 内容 API、Quizlet API
📊
monthly-reporter
月度进展报告生成
P2
字段内容
触发方式Cron 定时，每月第 25 天自动生成草稿
数据来源飞书文档中当月所有反馈记录、课节数、作业完成情况
报告内容 1本月上课节数、出勤率、作业完成率（量化数据）
报告内容 2老师评价关键词趋势（进步点 / 持续关注点）
报告内容 3结合长远目标（UWC 面试 / 雅思 7.5）的阶段性评估
报告内容 4下月学习重点预告（提前与教师确认）
人工介入Carol 审核 + 补充个性化细节后发送
依赖接口飞书 API（数据读取）、LLM API（报告撰写）、企业微信
🧮
teacher-hours-tracker
教师课时核算与报送
P3
字段内容
触发方式Cron 定时，每月最后一天自动汇总
数据来源飞书多维表格课表中的实际授课记录
核算对象外教 Ryan（课时数 × 外教单价）+ 中教 Vincent（课时数 × 中教单价）
输出格式Excel 课时统计表（教师姓名、课节日期、课时数、单价、应付金额）
发送对象飞书消息发至【教师管理团队】群，附 Excel 附件
人工介入Carol / 管理团队确认无误后提交财务
依赖接口飞书 API（课表读取）、Excel 生成库、飞书消息 API
lesson-reminder 提醒节点拆解
提醒节点 触发时间 接收对象 消息示例 触发条件
T-1 天提前提醒 上课前一天 09:00 微信服务群（家长） "Ray 妈妈，明天晚上 21:00 是 Ryan 老师的课哦，课前资料已发在群里 ✅" 仅前三节课
T-1 天资料推送 上课前一天 20:00 微信服务群 推送 Unlock 课件 / 预习词汇表，通知家长查收 所有课程
T-0 课前提醒 上课当天，提前 2 小时 微信服务群（学生+家长） "Ray，今晚 21:00 上课，记得提前 5 分钟测试设备哦 🎧 Zoom：[link]" 所有课程

💡 核心判断逻辑：读取学员档案「已上课节数」→ ≤3 节走高频双重提醒；>3 节仅走 T-0 + T-1 资料推送

三、自动化事件流总览
触发事件 触发源 调用 Skill 后续动作链
🆕 新学员签约 Carol 手动 student-onboarding 建档 → 创建飞书文件夹 → 发 Zoom 链接 → 通知外教/中教
⏰ 每天 08:00 扫描 Cron 定时 lesson-reminder 扫描当日/次日课表 → 判断课节数 → 发对应提醒
📚 每天 20:00 Cron 定时 lesson-reminder 从飞书 01_课前资料 读取 → 推送次日课前资料到微信群
🎬 Zoom 课程结束 Zoom Webhook zoom-recorder-manager 下载录像 → 企业云盘备份 → 飞书备份 → 发回放链接到微信群
📋 老师提交反馈 飞书表单 feedback-processor AI 提炼 → Carol 审核 → 发微信群 → 归档飞书
⚠️ 反馈过于笼统 feedback-processor 检测 ai-learning-advisor 生成个性化学习建议 → Carol 审核 → 合并进反馈发送
📅 每月第 25 天 Cron 定时 monthly-reporter 汇总飞书数据 → AI 起草月报 → Carol 审核 → 发家长
🗓️ 每月最后一天 Cron 定时 teacher-hours-tracker 汇总课时 → 生成 Excel → 发教师管理团队群
四、飞书多维表格数据底座
数据表名 核心字段 被哪些 Skill 使用 读/写
学员档案表 学员ID、姓名、年级、目标、外教、中教、上课时间、微信群ID、已上课节数 所有 Skill 初始写入 / 每节课更新
课表记录表 课节ID、学员ID、上课日期、外教、中教、教材单元、状态（待上/已上/已取消） lesson-reminder、teacher-hours-tracker 每次上课前写入 / 课后更新状态
反馈记录表 课节ID、反馈来源、原始文本、AI提炼表扬点、AI提炼改进点、Carol审核状态、发送时间 feedback-processor、monthly-reporter 每节课后写入 / 月度汇总读取
资料库表 资料ID、课节ID、类型（课前/课后）、内容、来源（AI生成/老师提供）、发送状态 ai-learning-advisor、lesson-reminder AI 生成时写入 / 推送时读取
课时核算表 月份、教师姓名、实际课时数、单价、应付金额、核算状态 teacher-hours-tracker 月末写入 / 财务确认后更新状态
五、实施优先级路线图
实施阶段 Skill 预计工期 核心价值 优先级
第一批 · 立刻上线 lesson-reminder 3 天 消除漏发提醒的最大风险，确保课前万无一失 P0
第一批 · 立刻上线 zoom-recorder-manager 3 天 保障录像不丢失，家长随时可查，提升信任感 P0
第二批 · 第二周 student-onboarding 2 天 建档标准化，一劳永逸，后续所有 Skill 的数据来源 P1
第二批 · 第二周 feedback-processor 5 天 核心交付价值，大幅降低 Carol 每节课的重复劳动 P1
第三批 · 第三周 ai-learning-advisor 4 天 提升服务质量，AI 补位笼统反馈，体现 VVIP 差异化 P2
第四批 · 月底前 monthly-reporter 4 天 提升家长满意度，体现长期规划价值，减少家长焦虑 P2
第四批 · 月底前 teacher-hours-tracker 2 天 内部管理规范化，工资准时准确发放，减少教务纠纷 P3
核心服务理念：极致的跟进 · 专业的数据沉淀 · 有温度的家校沟通｜VVIP 客户买的不仅是课程，更是「被懂」的感觉和顶级教育规划的安心感。所有自动化设计以「永远比家长多想一步」为准则——在家长来催要回放或反馈之前，我们已经主动发在群里了。
