# VVIP 学员服务 SOP 自动化设计全览

## 一、Skill 模块总览

| # | Skill 名称 | 对应 SOP 阶段 | 触发方式 | 优先级 | Carol 是否介入 |
|---|-----------|-------------|---------|-------|--------------|
| 1 | `student-onboarding` | 第一阶段：正式课建档与启动 | 手动触发（新学员签约时） | P1 | ✅ 首次需人工初始化 |
| 2 | `lesson-reminder` | 第二阶段 + 第三阶段：提醒 | Cron 定时（每天 08:00 扫描课表） | P0 | ❌ 全自动 |
| 3 | `zoom-recorder-manager` | 第三阶段：视频备份 | Zoom Webhook（课程结束事件） | P0 | ❌ 全自动 |
| 4 | `feedback-processor` | 第三阶段：课后反馈处理 | 飞书表单提交 / 老师填写反馈后 | P1 | ✅ 审核后发送 |
| 5 | `ai-learning-advisor` | 第二阶段：AI 智能补位 | 由 `feedback-processor` 调用 | P2 | ✅ 审核资料 |
| 6 | `monthly-reporter` | 第四阶段：月度家校沟通 | Cron 定时（每月最后一周） | P2 | ✅ 审核后发送 |
| 7 | `teacher-hours-tracker` | 第四阶段：教务核算 | Cron 定时（每月末） | P3 | ✅ 确认后报送 |

---

## 二、各 Skill 详细设计

### Skill 1 — `student-onboarding` 学员建档

| 字段 | 内容 |
|------|------|
| **触发方式** | 手动触发，新学员签约后执行一次 |
| **输入数据** | 学员姓名、年级、长远目标、外教/中教姓名、上课时间、微信群 ID |
| **动作 1** | 调用 Zoom API → 创建学员专属固定会议链接 |
| **动作 2** | 调用飞书 API → 创建专属文件夹（`01_课前资料 / 02_课程回放 / 03_老师反馈与作业`） |
| **动作 3** | 初始化学员档案（飞书多维表格写入基础信息） |
| **动作 4** | 将 Zoom 链接 + 上课操作指南发送至微信服务群 |
| **动作 5** | 分别发送确认消息给外教 Ryan 和中教 Vincent（包含上课时间确认） |
| **输出** | 飞书档案链接 + 微信群确认消息 |
| **依赖接口** | Zoom API、飞书 Open API、企业微信群机器人 |

---

### Skill 2 — `lesson-reminder` 课程提醒

| 提醒节点 | 触发时间 | 接收对象 | 消息内容 | 特殊逻辑 |
|---------|---------|---------|---------|---------|
| **T-1 天提前提醒** | 上课前一天 09:00 | 微信服务群（家长） | "Ray 妈妈，明天晚上 21:00 是 Ryan 老师的课哦，课前资料已发在群里 ✅" | 仅前三节课触发 |
| **T-0 课前提醒** | 上课当天，提前 2 小时 | 微信服务群（学生+家长） | "Ray，今晚 21:00 上课，记得提前 5 分钟测试设备哦 🎧 Zoom 链接：[link]" | 所有课程触发 |
| **T-1 天资料提醒** | 上课前一天 20:00 | 微信服务群 | 推送课前预习资料（Unlock 课件 / 词汇表）并通知家长查收 | 从飞书 `01_课前资料` 读取 |

| 字段 | 内容 |
|------|------|
| **核心判断逻辑** | 读取学员档案中的「已上课节数」→ ≤3 节：高频双重提醒；>3 节：仅 T-0 提醒 |
| **依赖接口** | 课表数据库（飞书多维表格）、企业微信群机器人 |

---

### Skill 3 — `zoom-recorder-manager` 录像管理

| 字段 | 内容 |
|------|------|
| **触发方式** | Zoom Webhook 事件：`recording.completed` |
| **动作 1** | 自动下载课程回放视频（MP4 格式） |
| **动作 2** | 上传至企业共享云盘（按学员/日期归档） |
| **动作 3** | 上传至学员飞书专属文件夹 `02_课程回放`（方便家长随时查阅） |
| **动作 4** | 发送回放链接到微信群，附注本节课主题和时长 |
| **动作 5** | 更新飞书多维表格课节记录（标记"已备份"） |
| **错误处理** | 下载失败时发飞书消息通知 Carol 人工处理 |
| **依赖接口** | Zoom Webhook + Zoom API、飞书 API、企业云盘 API、企业微信 |

---

### Skill 4 — `feedback-processor` 反馈处理

| 字段 | 内容 |
|------|------|
| **触发方式** | 老师提交飞书反馈表单后自动触发 |
| **输入** | 外教/中教原始反馈文本（英文或中文） |
| **AI 处理步骤 1** | 提炼结构化内容：**表扬点**（具体化、价值化表述） |
| **AI 处理步骤 2** | 提炼结构化内容：**改进点 + 解决方案**（必须带方案，不能只说问题） |
| **AI 处理步骤 3** | 专业术语翻译：`"needs structure"` → `"需要建立应对国际学校面试的逻辑框架"` |
| **AI 处理步骤 4** | 判断反馈是否过于笼统（如仅 "needs more vocabulary"）→ 若是，自动调用 `ai-learning-advisor` |
| **人工介入** | Carol 在飞书审批流中审核 AI 生成内容，确认无误后点击「发送」 |
| **动作（审核通过后）** | 排版后发送至微信服务群 |
| **动作（同步归档）** | 将结构化反馈写入飞书 `03_老师反馈与作业` 文件夹 |
| **依赖接口** | 飞书表单 API、LLM API（反馈提炼）、飞书审批 API、企业微信 |

---

### Skill 5 — `ai-learning-advisor` AI 学习建议

| 字段 | 内容 |
|------|------|
| **触发方式** | 由 `feedback-processor` 检测到笼统反馈后自动调用 |
| **输入** | 学员档案（年级/目标/当前教材进度）+ 老师的笼统反馈内容 |
| **AI 生成内容 1** | BBC 6分钟英语推荐（匹配本节课主题，附推荐理由） |
| **AI 生成内容 2** | Quizlet 词卡推荐 或 自动生成专属词汇表 |
| **AI 生成内容 3** | 阅读/听力扩展材料（匹配 G7 水平 + UWC 备考方向） |
| **AI 生成内容 4** | 给学生的个性化一句话推荐语（如："Ray，这期 BBC 是关于 AI 的，你一定感兴趣 🤖"） |
| **人工介入** | Carol 审核资料内容，确保专业准确 |
| **输出** | 排版好的学习资料卡片，合并进 `feedback-processor` 的发送内容 |
| **依赖接口** | LLM API、BBC/Quizlet 内容 API |

---

### Skill 6 — `monthly-reporter` 月度报告

| 字段 | 内容 |
|------|------|
| **触发方式** | Cron 定时，每月第 25 天自动生成草稿 |
| **数据来源** | 飞书文档中当月所有反馈记录、课节数统计、作业完成情况 |
| **报告内容 1** | 本月上课节数、出勤率、作业完成率（量化数据） |
| **报告内容 2** | 老师评价关键词趋势（进步点 / 持续关注点） |
| **报告内容 3** | 结合长远目标的阶段性评估（UWC 面试准备进度 / 雅思 7.5 差距分析） |
| **报告内容 4** | 下月学习重点预告（与中教/外教提前沟通确认） |
| **话术草稿示例** | "Ray 妈妈，这个月在 Ryan 老师辅导下，Ray 在口语逻辑'五步法'进步显著，下月我们重点攻克学术阅读词汇量……" |
| **人工介入** | Carol 审核 + 补充个性化细节后发送 |
| **输出渠道** | 微信群发送 + 飞书文档归档 |
| **依赖接口** | 飞书 API（数据读取）、LLM API（报告撰写）、企业微信 |

---

### Skill 7 — `teacher-hours-tracker` 教师课时核算

| 字段 | 内容 |
|------|------|
| **触发方式** | Cron 定时，每月最后一天自动汇总 |
| **数据来源** | 课表数据库（飞书多维表格）中的实际授课记录 |
| **核算对象** | 外教 Ryan（课时数 × 外教单价） + 中教 Vincent（课时数 × 中教单价） |
| **输出格式** | Excel 课时统计表（含：教师姓名、课节日期、课时数、单价、应付金额、备注） |
| **发送对象** | 飞书消息发送至【教师管理团队】群，附 Excel 附件 |
| **人工介入** | Carol/管理团队确认无误后提交财务 |
| **依赖接口** | 飞书 API（课表数据读取）、Excel 生成库、飞书消息 API |

## 三、基础设施：飞书多维表格数据底座

| 数据表 | 核心字段 | 用途 |
|-------|---------|------|
| **学员档案表** | 学员ID、姓名、年级、目标、外教、中教、上课时间、微信群ID、已上课节数 | 所有 Skill 的数据来源 |
| **课表记录表** | 课节ID、学员ID、上课日期、外教、中教、教材单元、状态（待上/已上/已取消） | `lesson-reminder` / `teacher-hours-tracker` 数据源 |
| **反馈记录表** | 课节ID、反馈来源（外教/中教）、原始文本、提炼后表扬点、提炼后改进点、Carol审核状态、发送时间 | `feedback-processor` 写入/读取 |
| **资料库表** | 资料ID、课节ID、类型（课前资料/课后拓展）、内容、发送状态 | `ai-learning-advisor` / `lesson-reminder` 读取 |
| **课时核算表** | 月份、教师姓名、实际课时数、单价、应付金额、核算状态 | `teacher-hours-tracker` 写入 |

## 四、自动化事件流总览

| 事件 | 触发源 | 调用 Skill | 后续动作 |
|-----|-------|-----------|---------|
| 新学员签约 | Carol 手动 | `student-onboarding` | 建档 → 发链接 → 通知教师 |
| 每天 08:00 | Cron | `lesson-reminder` | 扫描当日/次日课表 → 发提醒 |
| 每天 20:00 | Cron | `lesson-reminder` | 推送次日课前资料 |
| Zoom 课程结束 | Zoom Webhook | `zoom-recorder-manager` | 下载 → 双备份 → 发回放链接 |
| 老师提交反馈 | 飞书表单 | `feedback-processor` | AI提炼 → Carol审核 → 发微信 → 归档 |
| 反馈过于笼统 | `feedback-processor` 检测 | `ai-learning-advisor` | 生成学习建议 → Carol审核 → 合并发送 |
| 每月第 25 天 | Cron | `monthly-reporter` | 汇总数据 → AI起草 → Carol审核 → 发月报 |
| 每月最后一天 | Cron | `teacher-hours-tracker` | 汇总课时 → 生成Excel → 发教管群 |

## 五、实施优先级路线图

| 阶段 | Skill | 预计工期 | 核心价值 |
|-----|-------|---------|---------|
| **第一批（立刻上线）** | `lesson-reminder` | 3 天 | 消除漏发提醒的最大风险 |
| **第一批（立刻上线）** | `zoom-recorder-manager` | 3 天 | 保障录像不丢失，家长随时可查 |
| **第二批（第二周）** | `student-onboarding` | 2 天 | 建档标准化，一劳永逸 |
| **第二批（第二周）** | `feedback-processor` | 5 天 | 核心交付价值，大幅降低 Carol 重复劳动 |
| **第三批（第三周）** | `ai-learning-advisor` | 4 天 | 服务质量提升，AI 补位笼统反馈 |
| **第四批（月底前）** | `monthly-reporter` | 4 天 | 提升家长满意度，体现长期规划价值 |
| **第四批（月底前）** | `teacher-hours-tracker` | 2 天 | 内部管理规范化，工资准时发放 |
